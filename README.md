# CS-405 Secure Software Portfolio (C++)

This repository contains my secure software engineering portfolio completed as part of **CS-405: Secure Coding** at Southern New Hampshire University (SNHU). The portfolio demonstrates secure coding principles implemented primarily in **C++ (C Plus Plus)**, along with supporting documentation, testing, and security policy development.

The goal of this portfolio is to show that security is not an afterthought, but an integrated part of the software development lifecycle.

---

## Portfolio Overview

This portfolio showcases practical applications of secure coding standards, vulnerability mitigation techniques, risk assessment, zero trust principles, and security governance. Each section aligns with course outcomes and highlights defensive programming practices designed to reduce software vulnerabilities.

---

## Portfolio Contents

### 🔍 Static Code Analysis
**Folder:** `Static Code Analysis/`  
Demonstrates the use of static analysis techniques to identify potential security flaws early in the development process.

Key concepts:
- Shift-left security
- Early vulnerability detection
- Secure development workflows

---

### 🔐 Encryption
**Folder:** `Encryption/`  
Explores encryption techniques and secure data handling practices, including protecting data at rest and in transit.

Key concepts:
- Confidentiality
- Secure storage
- Cryptographic best practices

---

### 🧪 Unit Testing
**Folder:** `Unit Testing/`  
Demonstrates how unit testing can be used to validate secure behavior, prevent regressions, and verify defensive coding practices.

Key concepts:
- Defensive testing
- Verification of security controls
- Exception safety

---

### ⚠️ Exception Handling
**Folder:** `Exceptions Handling/`  
Focuses on secure exception handling to ensure predictable program behavior and prevent crashes or data leaks.

Key concepts:
- Fail-safe behavior
- Robust error handling
- Resource management

---

### 🧠 Memory Safety and Buffer Overflow Prevention
**Folder:** `Memory Safety/`  
Demonstrates techniques to prevent memory-related vulnerabilities such as buffer overflows and unsafe pointer usage in C++.

Key concepts:
- Memory management
- Secure use of pointers
- Prevention of common C/C++ vulnerabilities

---

### 💉 Injection Prevention (SQL Injection)
**Folder:** `Injection Prevention/`  
Shows how improper input handling can lead to injection vulnerabilities and how secure coding practices mitigate these risks.

Key concepts:
- Input validation
- Parameterized queries
- Trust boundary enforcement

---

### 📜 Security Policy and Governance
**Folder:** `Security Policy/`  
Contains a secure software development policy and supporting presentation outlining organizational security standards and practices.

Key concepts:
- Secure coding standards
- Risk assessment
- Authentication, authorization, and accountability (AAA)
- Policy-driven security

---

### 📝 Reflection
**Folder:** `Reflection/`  
Includes reflective writing discussing secure coding standards, risk-based decision making, zero trust architecture, and security policy implementation.

Key concepts:
- Critical reflection
- Secure software design
- Zero trust security model

---

## Core Security Principles Demonstrated

- Secure coding standards (OWASP, SEI CERT)
- Risk assessment and cost-benefit analysis
- Zero trust (“never trust, always verify”)
- Defense in depth
- Secure software development lifecycle (SSDLC)
- Security policy implementation

---

## Conclusion

This portfolio demonstrates a comprehensive approach to secure software development using C++. By integrating secure coding standards, risk evaluation, zero trust principles, and governance policies, the artifacts in this repository show how security can be embedded throughout the development lifecycle rather than applied after deployment.
